Software Design Specification

1.Introduction

"IntProf"is a platform for teacher and students. It's a platform for doing live classes, sharing educational resources, using as Q&A platform, taking attendance and so on. It's going to work as a bridge between teachers and students. In this document we will be covering all the design procedures required for this software implementation

2. Architectural Design

In this whole project we will follow MVC (MODEL-VIEW-CONTROLLER) architectural pattern to implement it. That means it is going to be a structured software design where we are going to implement all our tasks and functions using Codeigniter Framework and where all the design patterns and functioanl procedures will follow the MVC architectural pattern. Below is a brief description about how Model, View and Controller going to work

View:

All the user views and interfaces will reside here. That means the frontend operations will be represented and done through view.

Controller:

All the backend operations (i.e: processing) will be done through controller classes. View classes will interact with controller classes to get processed

Model:

All the database queries will be stalled here, and controller classes will interact with model classes for pulling or pushing data into database

3.Component Design

3.1 Website consists of many components. Components are related and depended to each other. An overall description of these components are given below:

Website

In our class diagram Website is the main class. The whole class diagram is based on our website class. There is a class of web user on which our main class Website is depended. For any sort of transaction users will have to go to the website. Then they will be able to check or post. So this is why website main class is depended upon web users, teacher and students.

Teacher

The Techer class is one of the primary users of the website. The main class Website is depended in a relation of 1 to many with other classes. This is because Without Web user -Teacher, the website will not be useful or utilized.

Students

Students are dependent on the Teacher for being able to use the website. Student class is related with the other classes. This is because Students will make will/can use the features of the website so that an analysis of their work can be at the end be graded. The relation between them is 1 to many.

Sign Up

Users, teachers and students, must sign up to the website in order to use it. Credentials including Username and Password must be given along with CID, TID and SID.

Sign In

Teacher and Student class will have to pass this in order to access website features.

Live Video Classroom

Teacher sets the date for the video class. Cid is used to schedule it. As a result of using CID, the TID and SID are also obtained. Live video class have VID and VNAME.

Q&A Platform

Teacher and Students posts here against their TID or SID and each question has a QID.

Posting & Submitting Assignments

Teacher posts Assignments each having an AID and ANAME. Students submit assignment against the AID and SID.

Posting Grades

At the end of term/semester, Grades are automatically generated based on performance with the teacher having the function to scale the grades against students’ SID.

Live Standing

This section updates automatically after each assignment submission, live video class and attendance and shows a statistical view against the SID of the students.

Online Attendance

The teacher can insert the attendance for students and also automatic attendance occurs as students join live class. Attendance occurs against SID and CID for a classroom.

Voice Search

Done using Artificial Intelligence as search is done by user’s voice commands. TID or SID of the user is kept for who is searching what.

Open-Closed Principle (OCP) – class should be open for extension but closed for modification will be the class based Principle used for this website.
