Software Requirements Specification

1.Introduction

1.1 Purpose

The name of the software which software requirements are specified in this document is "IntProf", which will act as a dynamic Online Teaching Assistant (Online TA) system. Current release number is 1.0.0.

1.2 Document Conventions

  • Font family: Times New Roman
  • Font Size: 11
  • Line spacing: 1.15

We used BOLD characters to highlight many important words. We have also used bullets and numbering to enlist different ordered and unordered list.

1.3 Intended Audience and Reading Suggestions

The document is mainly intended for 3 kinds of people. They are:

  • Project Executives (who are going to construct the whole system in many stages, i.e. Developers, project manager, testers, marketing staffs and documentation writer)
  • Users (Teachers, Students)
  • System Administrators (who are going to administrate the system after when it runs)
For all of the readers the Introduction section is very important. Because everyone is going to get a glimpse of what this project is intended for, what is the usability of this and system and what is the main characteristics of this system. For the Developers and other Project Executives Section 2, 3, 4, 5 are very important as the main project scopes, detailed requirements and other related specifications are sketched there. For the marketing staffs Section 1, 5 are the most important part for gathering marketing hints. Section 4: System Features is important for all the audience as this part describes in details what the features of the system are but if anyone is only intending for knowing the product features in brief then Section 1.4 will be very helpful.

1.4 Product Scope

This website is going to be worked as a complete system where teacher and students will be connected virtually. Teachers can create classes and students can enroll themselves after a process of verification. There will be scope for live classes through video sharing. It will also work as a Q&A platform among students and teachers. Teachers can post assignments and students can also submit their works online. This platform will be also used for posting grades and/or marks. A student can check out his/her live standing/performance in different classes. We are hoping to launch online attendance system in every classes.

2. Overall Description

This section is a complete overview of the entire system. +This section is about the functionalities, dependencies, constrains, assumptions of the product. It also describe the various user classes that we assume will use this product, how the system interacts with the user.

2.1 Product Perspective

Online Teaching Assistant (Online TA) system is a web based application designed to minimize the amount of time required for data management and operation on the system. This product is a new self contend product. This system should be able to find a route using user criteria. User can login to his or her account and see what they are looking for. The system will show the grades, the regular attendance, and Q/A sections. Assignments topic also can be posted and receive. There?ll be the option for live classes.

2.2 Product Functions

The system comprises of 2 major modules with their sub-modules as follows:

Teacher:

  • Teacher will login on his personal account. They will create Classes, name of the Classes, Add Image
  • Teacher will post assignments and can check them
  • Teacher can take online video classes
  • Teachers will take regular attendance and take participate in Q/A box.

Student:

  • Student can register his personal detail, and login his/her account.
  • After login student will be able to view the Q/A box, attendance.
  • Student will check the due assignments and they can submit the attendance.
  • Student will apply for late submission.
  • Student will participate in online class.
  • For the mobile applications we?ll add pre-class notification for student.
  • For mobile application student can check the exam dates.
  • For mobile application, student will get a welcome to the class message after each login.

2.3 User Classes and Characteristics

There will be 2types of users Admin and general users

Admin

It can pretty much do everything. Admin can login to his/ her account; they can add data, delete data or edit data based on the requirement. The admins can manage/modify information. They can see the user details, student feedback. Admins can logout from his account.

User

General users can use both the website and mobile app. They will have an account. Users can search, view, ask/reply questions in Q/A bar, take attendance and Take/attend online classes. They can?t change anything of the system. After login they will search according to their need, and the system will provide information.

The system is mainly for the Teachers and students. Teachers and students can use this website for information about the courses, assignments, attendance, marks that they are searching for. They can get all the information in details information. It can be helpful for other users too; those who wants to know the course .But the most important users are the teachers and students.

2.4 Operating Environment

  • Operating environment is as listed below.
  • Distributed database
  • Client/server system
  • Operating system: Windows, however, it will be a website, so will be able to run on any operating system
  • Database: MYSQL database
  • Platform: JavaScript, PHP

2.5 Design and Implementation Constraints

User should have internet connection. Without an internet connection, the users cannot log in to the system. So, they cannot interact with the system without internet.

Another constraint is data capacity. As we are not using our own data center, we are might face issue with capacity.

2.6 User Documentation

The system will provide a user manual along with the software and there will be a disk which contains video tutorial.

2.7 Assumptions and Dependencies

User interface and some functionalities can change during the development process of project. And also new functionalities can be added which is able to change the dependent system requirements.

If the phone does not have enough hardware resources available for the application, for example, the users might have allocated them with other applications; there may be scenarios where the application does not work as intended or even at all.

3. External Interface Requirements

3.1 User Interfaces

The website will have a straightforward and a clean look which easy to navigate. At first sight, a user will feel everything in the website is well organized and easy to explore. There will be two separate User Interfaces. One for the teacher while other for the student.

There will be graphical user interface (GUI) which will allow the users to provide feedback. The teachers can post grades here on real time and system will make a statistical standing and keep it updated. Errors will be displayed if user give wrong inputs. There will also be a help button which when the user presses will show information to the user about the page content and this button will appear on every page.

Front end Software: HTML, CSS and WordPress

Back end Software: PHP, Laravel Framework & MySQL

3.2 Hardware Interfaces

Any Device that is able to run a web browser which has the ability to process above mentioned Front end software and also the device has to meet the requirements of the browser to run without lagging.
