Software Requirements Specification

1.Introduction

1.1 Purpose

The name of the software which software requirements are specified in this document is "IntProf", which will act as a dynamic Online Teaching Assistant (Online TA) system. Current release number is 1.0.0.

1.2 Document Conventions

  • Font family: Times New Roman
  • Font Size: 11
  • Line spacing: 1.15

We used BOLD characters to highlight many important words. We have also used bullets and numbering to enlist different ordered and unordered list.

1.3 Intended Audience and Reading Suggestions

The document is mainly intended for 3 kinds of people. They are:

  • Project Executives (who are going to construct the whole system in many stages, i.e. Developers, project manager, testers, marketing staffs and documentation writer)
  • Users (Teachers, Students)
  • System Administrators (who are going to administrate the system after when it runs)
For all of the readers the Introduction section is very important. Because everyone is going to get a glimpse of what this project is intended for, what is the usability of this and system and what is the main characteristics of this system. For the Developers and other Project Executives Section 2, 3, 4, 5 are very important as the main project scopes, detailed requirements and other related specifications are sketched there. For the marketing staffs Section 1, 5 are the most important part for gathering marketing hints. Section 4: System Features is important for all the audience as this part describes in details what the features of the system are but if anyone is only intending for knowing the product features in brief then Section 1.4 will be very helpful.

1.4 Product Scope

This website is going to be worked as a complete system where teacher and students will be connected virtually. Teachers can create classes and students can enroll themselves after a process of verification. There will be scope for live classes through video sharing. It will also work as a Q&A platform among students and teachers. Teachers can post assignments and students can also submit their works online. This platform will be also used for posting grades and/or marks. A student can check out his/her live standing/performance in different classes. We are hoping to launch online attendance system in every classes. Users can search queries using voice search

2. Overall Description

2.1 Product Perspective

Online Teaching Assistant (Online TA) system is a web based application designed to minimize the amount of time required for data management and operation on the system. This product is a new self contend product. This system should be able to find a route using user criteria. User can login to his or her account and see what they are looking for. The system will show the grades, the regular attendance, and Q/A sections. Assignments topic also can be posted and receive. There will be the option for live classes.

2.2 Product Functions

The system comprises of 2 major modules with their sub-modules as follows:

Teacher:

  • Teachers will be able to login on their personal accounts. They will create Classes, name of the Classes, can add related resources
  • Teachers can post different notice & announcements
  • Teachers can post assignments and can check submissions
  • Teachers can take online video classes
  • Teachers will be able to take regular attendance and take participate in Q/A box.
  • They can schedule exams on calender
  • Teachers can post marks of different students and can grade them online
  • Teachers will be able to share study materials
  • They can search past notices and/or other announcements through voice search

Student:

  • Students can register his personal detail, and login his/her account.
  • After login student will be able to view the ask question box and they can check attendance.
  • Students will be able to check the due assignments and they can submit.
  • Students can apply for late submission.
  • Students can participate in online class.
  • Students can check exam dates and upcoming events on calender
  • Students will be able to see their current standing on each class. They can check their grades and marks
  • They can search past notices and/or other announcements through voice search

2.3 User Classes and Characteristics

There will be 2 types of users Admin and general user

Admin

He/She can pretty much do everything. Admin can login to his/ her account; they can add data, delete data or edit data based on the requirement. The admins can manage/modify information. They can see the user details, can define who is teacher and who is student. They can add, remove or block an account remotely

User

General users are mainly Teachers and Students. They can do their specified actions specified in Section 2.2.

2.4 Operating Environment

  • Operating environment is as listed below.
    • Distributed database
    • Client/server system
    • Operating system: Any, it will be a web based application, so will be able to run on any operating system
    • Database: MYSQL database
    • Platform: PHP, JavaScript, JQuery

2.5 Design and Implementation Constraints

User should have internet connection. Without an internet connection, the users cannot log in to the system. So, they cannot interact with the system without internet.

Another constraint is data capacity. As we are not using our own data center, we are might face issue with capacity.

2.6 User Documentation

The system will provide a user manual along with the software and there will be a disk which contains video tutorial.

2.7 Assumptions and Dependencies

User interface and some functionalities can change during the development process of project. And also new functionalities can be added which is able to change the dependent system requirements.

If the phone does not have enough hardware resources available for the application, for example, the users might have allocated them with other applications; there may be scenarios where the application does not work as intended or even at all.

3.External Interface Requirements

3.1 User Interfaces

The website will have a straightforward and a clean look which easy to navigate. At first sight, a user will feel everything in the website is well organized and easy to explore. There will be two separate User Interfaces. One for the teacher while other for the student.

There will be graphical user interface (GUI) which will allow the users to provide feedback. The teachers can post grades here on real time and system will make a statistical standing and keep it updated. Errors will be displayed if user give wrong inputs. There will also be a help button which when the user presses will show information to the user about the page content and this button will appear on every page.

Frontend Software: HTML, CSS and Bootstrap

Backend Software: PHP, Codeigniter Framework, JavaScript, JQuery & MySQL

3.2 Hardware Interfaces

Any Device that is able to run a web browser which has the ability to process above mentioned Front end software and also the device has to meet the requirements of the browser to run without lagging.

5.Other Non-functional Requirements

Here, we specify some non-functional constraints that the website satisfies in order to be more concrete and stable.

5.1 Performance Requirements

Registration: The website will require teachers and students to register online. They will have to submit certain credentials and will be verified once confirmed. Similar verification will be done when a student is assigned to a certain class.

Information: Student information from registration will be forwarded to the teacher and admin.

Live Class: Online Class interaction timing will depend on the internet connectivity and the speed. Probable chances of buffer/delay if internet speed is detected comparatively slower than usual.

5.2 Safety Requirements

In case of a server breakdown, the last checkpoint will be saved automatically.

5.3 Security Requirements

Every user of the website must log in in order to take part, and actions of the students will be monitored by the teacher. The files shared by the teacher must be limited to the viewer. Only students will be allowed to view the homeworks and assignments provided by the teacher. However, if the student posts his/her query in the QA platform, everyone will be able to view the post and take necessary measure. In that case, only teachers will be allowed to make any change to that post.

5.4 Software Quality Attributes

We will use PHP in order to avoid our website from crashing. While as, Bootstrap and Codeigniter framework will help us make the project an industry-standard website.

5.5 Business Rules

  • Students cannot change his/her grades/standings/attendance.
  • Student won’t be able to change the assignment/homework once submitted
  • Students cannot share files in-between themselves
  • Teacher-student confidentiality can be kept.

6. Other Requirements

The website will require to follow the legal requirements while being used such as US Digital Millennium Copyright Act. The project is sustainable and can be reused for similar purposes such as for online interaction purposes.
